# ベジ利き (vegekiki)

野菜・果物・魚など食材の選び方（目利きポイント）をすぐ確認できる無料Webサイトです。

サイトURL: https://vegekiki.jp/

## 概要

スーパーでの買い物中に「この野菜、どれを選べばいいのか」と迷う時間を短縮できるサービスです。食材名で検索したり、カテゴリで絞り込んだりして、新鮮な食材の見分け方をすぐに確認できます。

## 機能

- カテゴリ絞り込み — 野菜 / 果物 / 魚 / その他（きのこ・肉類）で表示を切り替え
- 食材名検索 — ひらがな・カタカナ・漢字に対応した検索
- 折り畳み機能 — 食材をタップすると画像を展開表示
- お気に入り登録 — ♡ボタンでお気に入り登録、リスト上部に固定表示
- レスポンシブ対応 — スマートフォン・タブレット・PCに対応

## 工夫した点

使いやすさと表示速度を重視して設計しました。

- **折り畳み表示** — 食材をタップしたとき別ページに遷移せず、その場でアコーディオン展開して目利きポイントを表示するようにした。スーパーの売り場でスムーズに確認できることを意識した
- **画像の軽量化** — 全画像をWebP形式で用意し、PNGと比べてファイルサイズを大幅に削減した。モバイル回線でも素早く表示されるよう配慮した
- **検索** — ひらがな・カタカナ・漢字のどれで入力しても検索できるよう、読み仮名マップとひらがな→カタカナ変換を組み合わせた
- **お気に入り登録** — LocalStorageを使い、ブラウザを閉じてもお気に入りが消えないようにした。登録済みの食材はリスト上部に固定表示されるようにした
- **スマホでの使いやすさ** — ヘッダーが隠れないよう固定し、リストエリアだけスクロールする構成にした

## 振り返り・今後の改善点

実際に公開・運用してみて気づいた課題です。

- **画像の視認性** — 目利きポイントを画像内に文字で入れたが、情報量が多くなりすぎて見づらくなった。テキストは画像の外に出し、画像はビジュアルに集中させる構成にすべきだった
- **比較画像のリアリティ** — 「良い状態」と「悪い状態」を対比する画像を用意したが、悪い状態が極端すぎて実際の売り場と乖離があった。日常的に目にする程度の差を示す画像の方が参考にしやすかった
- **想定と実際のユーザーの行動との相違** — スーパーに並んでいる食材はもともと状態のよいものが多く、アプリを見ながら選ぶメリットが自分の目で即座に判断するメリットを上回りにくい。この課題がリピート率の低下につながったと考えている。今後は「このサービスを作ることで、ユーザーの行動がどう変化し、そのようなメリットを生み出すのか」をもっと具体的に想定してから開発したい。
